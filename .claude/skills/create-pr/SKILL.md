# GitHub PR 作成スキル

あなたは GitHub Pull Request を作成するアシスタントです。
`.github/pull_request_template.md` のテンプレートをベースに、バックエンドリポジトリに最適化したPRを作成します。

## 設定

- リポジトリ: `kpool09122/kpool-backend`
- ベースブランチ: `$ARGUMENTS` が指定されていればそれを使用、なければ `main`
- テンプレート: `.github/pull_request_template.md`

## バックエンド固有の除外項目

以下のセクションはバックエンドリポジトリでは **不要なため省略** します：
- 📸 スクリーンショット・デモ
- 💄 UI/UX (Style) の変更種類チェック
- 動作確認（手動テスト記載欄）

## 実行手順

### Step 1: 現在の状態を把握

以下を並列で実行してください：

- `git status` で未コミットの変更を確認
- `git log` で現在のブランチのコミット履歴を確認
- `git diff <ベースブランチ>...HEAD` でベースブランチからの全差分を確認
- 現在のブランチがリモートにpush済みか確認

未コミットの変更がある場合はユーザーに報告し、コミットするか確認してください。

### Step 2: 差分を分析

コミット履歴と差分を読み、以下を整理してください：

- **変更内容の要約**: 何をしたか
- **変更の種類**: 新機能 / バグ修正 / リファクタリング / ドキュメント / テスト / ビルドCI / パフォーマンス / 削除
- **変更理由・背景**: なぜこの変更が必要か
- **レビューポイント**: 特に注目すべき点
- **注意事項**: 破壊的変更、マイグレーション等

### Step 3: 関連Issueの確認（必須）

ユーザーに以下を **必ず質問** してください：

> この PR に関連する Issue 番号を教えてください。
> - **Closes #**: この PR で完了する Issue（マージ時に自動クローズ）
> - **Fixes #**: この PR で修正するバグ Issue（マージ時に自動クローズ）
> - **Relates to #**: 関連するが完了はしない Issue
>
> 該当なしの場合は「なし」と回答してください。

### Step 4: PR本文を構成してユーザーに提示

テンプレートに沿ってPR本文を構成し、ユーザーに内容を提示して確認を取ってください。
タイトルは70文字以内で簡潔に。

### Step 5: PR作成

確認が取れたら以下を実行してください：

1. リモートにpushされていなければ `git push -u origin <ブランチ名>` で push
2. `gh pr create` でPRを作成
3. 作成したPRのURLをユーザーに報告

### PR本文フォーマット

```markdown
## 📝 変更内容

<!-- 差分分析から自動生成 -->

- [ ] 📚 ドキュメント (Documentation)
- [ ] 🧪 テスト (Tests)
- [ ] 🔨 ビルド/CI (Build/CI)

## 🎯 変更理由・背景

- [ ] `make check` を実行し、すべてのテストがパスすることを確認
- [ ] 新しく追加した機能に対するテストを作成
- [ ] 既存のテストが壊れていないことを確認

## 🔍 レビューのポイント

<!-- 差分分析から自動生成 -->

## 📖 関連情報

### 関連Issue・タスク

Closes #
Fixes #
Relates to #

## ⚠️ 注意事項

<!-- 破壊的変更、マイグレーション等があれば記載。なければ「特になし」 -->

---

## チェックリスト

- [ ] 自分でコードレビューを実施した
- [ ] 適切なブランチ名を使用している
- [ ] コミットメッセージが適切である
- [ ] 必要に応じてドキュメントを更新した
- [ ] 破壊的変更がある場合は適切に文書化した
```

## 重要な原則

- **関連Issueは必ず確認する**: ユーザーに質問せずにPRを作成しないこと
- **差分を実際に読む**: コミットメッセージだけでなく、コード差分を読んで正確な変更内容を記載する
- **該当しない変更種類のチェックは外す**: 該当するものだけ `[x]` にする
- **関連Issueが「なし」の場合**: `Closes #` / `Fixes #` / `Relates to #` の行は削除する
- **テンプレートの不要セクションは出力しない**: スクリーンショット、動作確認欄は含めない
